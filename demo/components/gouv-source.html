Composant invisible de connexion aux donnees. Recupere des donnees depuis une API REST et les distribue aux autres composants via le systeme d'evenements.

Invisible : Ce composant ne rend rien visuellement. Il agit comme un "pont" entre votre API et les composants de visualisation.

Attributs

AttributTypeDescription
idStringIdentifiant unique (requis). Les autres composants l'utilisent pour s'abonner.
urlStringURL de l'API (relative ou absolue)
methodStringMethode HTTP : GET ou POST
headersStringEn-tetes HTTP en JSON : {"Authorization": "Bearer xxx"}
paramsStringParametres de requete ou body POST en JSON
transformStringChemin JSON pour extraire les donnees : "data.items"
refreshNumberIntervalle de rafraichissement en secondes (0 = pas de refresh)
paginateBooleanActive la pagination serveur. Injecte page et page_size dans l'URL. Auto-extrait json.data quand aucun transform n'est defini.
page-sizeNumberTaille de page pour la pagination serveur (defaut : 20)

Exemples

Connexion simple a une API OpenDataSoft

<gouv-source
  id="prix-ct"
  url="https://data.economie.gouv.fr/api/explore/v2.1/catalog/datasets/prix-controle-technique/records"
  transform="results"
></gouv-source>

Avec parametres et en-tetes

<gouv-source
  id="api-privee"
  url="https://mon-api.gouv.fr/data"
  method="POST"
  headers='{"Authorization": "Bearer TOKEN"}'
  params='{"limit": 100, "filters": {"annee": 2024}}'
  transform="data.items"
  refresh="60"
></gouv-source>

Pagination serveur (API Tabular)

Avec paginate, chaque changement de page dans un gouv-datalist ou gouv-display en aval declenche un nouvel appel API avec ?page=N&page_size=M. Permet de naviguer dans des datasets de dizaines de milliers d'enregistrements sans tout charger. 

<gouv-source
  id="maires"
  url="https://tabular-api.data.gouv.fr/api/resources/2876a346-d50c-4911-934e-19ee07b0e503/data/"
  paginate
  page-size="20"
></gouv-source>

<!-- Le datalist detecte automatiquement la pagination serveur -->
<gouv-datalist source="maires"
  colonnes="Nom de l'elu:Nom, Prenom de l'elu:Prenom, Libelle du departement:Departement"
  pagination="20">
</gouv-datalist>

Auto-extraction : en mode paginate, pas besoin de transform="data". Les donnees sont automatiquement extraites depuis json.data et les metadonnees de pagination (json.meta) sont stockees pour calculer le nombre total de pages.

Evenements

Le composant emet des evenements personnalises que les autres composants peuvent ecouter :

  • gouv-data-loaded : Donnees chargees avec succes
  • gouv-data-loading : Chargement en cours
  • gouv-data-error : Erreur de chargement
