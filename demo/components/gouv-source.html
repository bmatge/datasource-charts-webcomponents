gouv-source

Composant invisible de connexion aux donnees. Recupere des donnees depuis une API REST et les distribue aux autres composants via le systeme d'evenements.

Invisible : Ce composant ne rend rien visuellement. Il agit comme un "pont" entre votre API et les composants de visualisation.

Attributs

AttributTypeDescription
idStringIdentifiant unique (requis). Les autres composants l'utilisent pour s'abonner.
urlStringURL de l'API (relative ou absolue)
methodStringMethode HTTP : GET ou POST
headersStringEn-tetes HTTP en JSON : {"Authorization": "Bearer xxx"}
paramsStringParametres de requete ou body POST en JSON
transformStringChemin JSON pour extraire les donnees : "data.items"
refreshNumberIntervalle de rafraichissement en secondes (0 = pas de refresh)

Exemples

1. Chargement simple

Charge des donnees depuis une API et extrait le champ "sites".

<gouv-source
  id="mes-donnees"
  url="/api/data.json"
  transform="sites">
</gouv-source>

2. Avec authentification

Ajoute un header d'authentification Bearer.

<gouv-source
  id="donnees-auth"
  url="/api/protected"
  headers='{"Authorization": "Bearer mon-token"}'
  transform="results">
</gouv-source>

3. Rafraichissement automatique

Recharge les donnees toutes les 30 secondes.

<gouv-source
  id="donnees-live"
  url="/api/metrics"
  transform="data"
  refresh="30">
</gouv-source>

4. Requete POST

Envoie une requete POST avec un body JSON.

<gouv-source
  id="recherche"
  url="/api/search"
  method="POST"
  params='{"query": "ministere", "limit": 50}'
  transform="results">
</gouv-source>

Evenements

Le composant emet des evenements personnalises que les autres composants peuvent ecouter :

  • gouv-data-loaded : Donnees chargees avec succes
  • gouv-data-loading : Chargement en cours
  • gouv-data-error : Erreur de chargement
